- **筛选条件：**

- **候选池：** 324 条
- **抽检数量：** 80 条

## 2. 抽样方法

采用**分层比例抽样**（Stratified Proportional Sampling）：

1. 按 L1 品类将 324 条 auto_pass 分为 19 个层
2. 每层按比例分配抽检名额（比例 = 层大小 / 324 × 80），最低保障 1 条
3. 层内采用两阶段选择：
   - 第一阶段：从每层内不同平台各取 1 条（保障平台多样性）
   - 第二阶段：余下名额从层内剩余样本随机抽取
4. 全局随机打乱排序

## 6-附. 人工抽检前发现并修正的系统输出问题

在抽检样本生成后、正式人工标注前，对 80 条样本进行了预检，发现并修正了 2 个系统输出错误：

### S782：店铺误作主实体

- **原始输出：** main_entity=店铺, L1=平台服务与账号服务 > 店铺与商家服务
- **问题：** 文本明确写"在淘宝网……店铺购买星巴克杯子，杯子漏水，盖子太紧，做工粗糙"。系统将购买渠道"店铺"当作投诉主实体，忽略了明确的商品词"星巴克杯子"
- **修正：** main_entity=杯子, L1=家居家装 > 厨房卫浴用品, confidence=0.85, need_human_review=FALSE
- **规则：** 当文本出现"在某店铺购买 + 明确商品词"时，商品词优先作为主实体；"店铺"只是购买渠道，不应覆盖商品实体
- **标注指南更新：** 已在 §14-附 新增"店铺购买 + 商品词"主实体判定规则

### S939：无来源实体（疑似幻觉）

- **原始输出：** main_entity=水性笔, L1=图书文娱与文创 > 文具与乐器, evidence_text=水性笔, classification_reason=小米水性笔投诉
- **问题：** 回查原文（153字 content_summary + 标题），全文无"水性笔"或"小米"。投诉为天猫超市半日达配送超时及赔付不公。"单笔订单"中的"笔"是量词非文具。系统凭空生成了"水性笔"和"小米"实体
- **修正：** main_entity=半日达配送, main_entity_type=PLATFORM_SERVICE, L1=平台服务与账号服务 > 物流配送服务, confidence=0.80, need_human_review=FALSE
- **规则：** 量词/单位词（单笔订单中的"笔"、两包尿不湿中的"包"）不应被识别为商品实体；禁止凭空生成原文不存在的实体词
- **标注指南更新：** 已在 §14 常见误匹配表中新增"单笔/多笔（订单量词）"条目

### 影响

- 两处修正均同步到主表 `stage1_pilot_category_results_500.csv` 和抽检文件 `stage1_manual_accuracy_sample_80.csv`
- manual 字段保持空白，不影响人工标注流程
- 主表 auto_pass 统计不变（S782 仍然 auto_pass，S939 仍然 auto_pass，仅分类修正）

## 7. 为什么这些样本适合做 auto_pass 准确率评估

- **全部来自 auto_pass**：这些样本被系统判定为置信度 ≥0.75 且无复核触发条件，实际准确率应该最高
- **已排除已知错误模式**：中文词边界误触发（包/猫/狗/成语）已三轮修复
- **无 taxonomy gap**：所有样本在 taxonomy 中均有匹配分类
- **L1 100% 覆盖**：19 个有 auto_pass 的 L1 全覆盖，评估结果有代表性
- **多平台覆盖**：18 个平台，避免单平台偏差
- **文本难度分层**：短文本（易）和中文本（需推理）混合，真实反映系统能力

## 8. 人工填写说明

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `manual_l1_correct` | TRUE/FALSE | L1 分类是否正确 |
| `manual_l2_correct` | TRUE/FALSE | L2 分类是否正确（L1 正确前提下判断） |
| `manual_final_l1_code` | string | 人工修正后的 L1 code（无需修正则留空） |
| `manual_final_l1_name` | string | 人工修正后的 L1 name |
| `manual_final_l2_code` | string | 人工修正后的 L2 code |
| `manual_final_l2_name` | string | 人工修正后的 L2 name |
| `manual_error_type` | string | 如分类有误，选择错误类型（见下方可选值） |
| `manual_comment` | string | 任何备注 |

### `manual_error_type` 可选值

| 值 | 含义 | 示例 |
|----|------|------|
| （留空） | 分类正确，无需修正 | — |
| `ENTITY_EXTRACTION_ERROR` | 商品/服务实体抽错或漏抽，导致分类错误 | "衣服"被抽成"N/A" |
| `L1_CLASSIFICATION_ERROR` | L1 大类选错 | 手机被归入家用电器 |
| `L2_CLASSIFICATION_ERROR` | L1 正确但 L2 选错 | 羽绒服→男装（应为女装） |
| `BRAND_CONFUSION` | 品牌辅助信号导致误判 | 苹果→食品生鲜（实际是 iPhone） |
| `PLATFORM_SERVICE_CONFUSION` | 平台服务和商品投诉混淆 | 运费→商品投诉（实际投诉运费险本身） |
| `TAXONOMY_GAP` | taxonomy 中没有合适分类 | 即使 gap=FALSE，也可能发现缺类目 |
| `AMBIGUOUS_TEXT` | 文本本身模糊/信息不足，难以判断 | "要求退款"无商品信息 |
| `OTHER` | 其他未列出的错误类型 | 在 comment 中说明 |

### 填写步骤

1. 阅读 `title` 和 `content_summary`
2. 判断系统给出的 `l1_name` / `l2_name` 是否正确
3. 如正确：`manual_l1_correct=TRUE`, `manual_l2_correct=TRUE`，其余 manual 字段留空
4. 如 L1 错：`manual_l1_correct=FALSE`，填写 `manual_final_l1_*` 和 `manual_error_type`
5. 如 L2 错（L1 对）：`manual_l1_correct=TRUE`, `manual_l2_correct=FALSE`，填写 `manual_final_l2_*`
6. 如有任何疑问，在 `manual_comment` 中记录

## 9. 准确率计算方法

完成 80 条人工标注后，按以下公式计算：

### L1 Accuracy
```
L1_accuracy = COUNT(manual_l1_correct = TRUE) / 80
```

### L2 Accuracy
```
L2_accuracy = COUNT(manual_l2_correct = TRUE) / 80
```
注：L1 错误的样本中 L2 必然错误（`manual_l2_correct = FALSE`）。

### Entity Extraction Error Rate
```
entity_error_rate = COUNT(manual_error_type = 'ENTITY_EXTRACTION_ERROR') / 80
```

### Auto-pass Error Rate（系统错误率）
```
auto_pass_error_rate = 1 - COUNT(manual_l1_correct = TRUE AND manual_l2_correct = TRUE) / 80
```
即：被系统判定为"自动通过"的样本中，实际有 L1 或 L2 错误的占比。

### 各类错误分布
按 `manual_error_type` 分组统计，识别系统的主要薄弱点。

### 期望目标
- L1 accuracy ≥ 80%
- L2 accuracy ≥ 70%
- entity extraction error rate ≤ 15%

低于目标则需要回溯 review cases 或进一步修复规则。
