# 品牌映射与归一化设计

## 0. 文档说明

本文档定义品牌识别与归一化模块的设计方案。品牌是项目中的**辅助维度**，用于品牌聚合分析、品牌风险监控和品牌投诉量统计，不能替代商品/服务品类分类。

**核心原则：**
- 品牌回答"属于哪个品牌/品牌集团"
- 商品品类回答"是什么商品/服务"
- 品牌只能辅助品类判断，不能覆盖品类判断

---

## 1. 品牌词典设计目标

1. **品牌归一化：** 将中英文、大小写、别名、产品线/系列名统一映射到标准品牌名
2. **品牌-品类关联：** 记录每个品牌常见的商品品类，辅助品类判断
3. **品牌歧义消解：** 标记具有歧义的品牌词，提供消歧规则
4. **品牌集团聚合：** 记录品牌集团归属，支持集团维度的投诉分析
5. **可扩展：** 支持从投诉数据中持续发现新品牌候选

---

## 2. 品牌主表字段设计

### 2.1 核心字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `brand_id` | String | 是 | 品牌唯一ID，格式 `BRAND_{UPPER_NAME}`，如 `BRAND_APPLE`、`BRAND_HUAWEI` |
| `normalized_brand_name` | String | 是 | 归一化品牌显示名，格式 `English名/中文名`，如 `Apple/苹果`、`小米/Xiaomi` |
| `chinese_name` | String | 是 | 品牌中文名（常用称呼），如 `苹果`、`华为` |
| `english_name` | String | 是 | 品牌英文名（官方名称），如 `Apple`、`HUAWEI` |
| `brand_group` | String | 否 | 品牌集团/母公司，如 `Apple Inc.`、`小米集团`、`华为技术有限公司` |
| `country_or_region` | String | 否 | 品牌所属国家/地区，如 `美国`、`中国`、`韩国` |
| `industry` | String | 否 | 品牌所属行业，如 `消费电子`、`运动服饰`、`家电`、`美妆` |

### 2.2 品类关联字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `common_categories` | Array | 是 | 品牌常见商品品类列表。每个品类包含 `l1_code`、`l1_name`、`l2_code`、`l2_name`、`frequency`（HIGH/MEDIUM/LOW） |
| `is_single_category_brand` | Boolean | 是 | 是否为单一品类品牌（如NVIDIA主要在数码电子） |
| `category_inference_confidence` | Float | 否 | 当只有品牌没有商品词时，通过品牌推断品类的默认置信度（0-1）。跨品类品牌应为低值 |

### 2.3 别名与产品线字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `aliases` | Array | 是 | 品牌别名列表：中文名、英文名、大小写变体、常见错拼、俗称 |
| `product_lines` | Array | 是 | 产品线/系列名列表。如 Apple 的 `["iPhone", "iPad", "MacBook", "AirPods", "Apple Watch", "iMac"]` |
| `sub_brands` | Array | 否 | 子品牌列表。如小米的 `["Redmi/红米", "POCO", "米家"]` |
| `historical_names` | Array | 否 | 历史/旧品牌名（如品牌更名前的名称） |

### 2.4 歧义与审核字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `ambiguity_level` | String | 是 | 品牌歧义程度：`LOW`（基本无歧义）、`MEDIUM`（需注意上下文）、`HIGH`（高度歧义，必须结合上下文） |
| `ambiguity_description` | String | 否（HIGH时强烈建议） | 歧义说明，如"苹果可能指水果，也可能指Apple品牌" |
| `disambiguation_rules` | Array | 否（HIGH时强烈建议） | 消歧规则列表，每条包含 `context_keywords`、`verdict`、`confidence_adjustment` |
| `review_required_when_brand_only` | Boolean | 是 | 当投诉中只有该品牌没有商品词时，是否必须人工复核 |
| `notes` | String | 否 | 其他注意事项 |

## 3. 品牌别名映射规则

### 3.1 别名类型

| 别名类型 | 说明 | 示例 |
|---------|------|------|
| `official_name` | 官方品牌名 | Apple、HUAWEI |
| `chinese_name` | 中文名 | 苹果、华为 |
| `english_name` | 英文名（含大小写变体） | apple、APPLE、huawei |
| `product_line` | 产品线/系列名（可独立识别为品牌） | iPhone、iPad、Mate、Pura |
| `sub_brand` | 子品牌名 | Redmi、红米、米家 |
| `alias` | 常见别名/俗称 | 果子→苹果、三叶草→Adidas |
| `abbreviation` | 品牌缩写 | HW→华为、XM→小米 |
| `common_misspelling` | 常见错拼 | Nvida→NVIDIA、Samsng→Samsung |

### 3.2 别名匹配优先级

当同一个原始文本匹配到多个别名时：

1. **精确匹配优先：** 完全匹配 > 模糊匹配
2. **产品线优先：** 如果匹配到产品线（如"iPhone"），通过产品线→母品牌映射，而非直接匹配母品牌别名
3. **长短优先：** 长别名匹配优先于短别名（"Apple Watch" 优先于 "Apple"）
4. **上下文验证：** 匹配后通过上下文商品词验证合理性

| 条件 | 可推断性 | 置信度 | 是否需要人工复核 |

## 6. 品牌歧义处理规则

### 6.1 日常用语歧义

| 品牌词 | 歧义含义 | 消歧关键词（品牌） | 消歧关键词（非品牌） |
|--------|---------|-----------------|------------------|
| 苹果 | 水果 | 手机、电脑、耳机、平板、手表、iOS、iPhone | 水果、坏了、不甜、一斤、新鲜 |
| 小米 | 谷物 | 手机、电视、电器、智能、系统、MIUI | 粮食、煮粥、一斤、杂粮、黄小米 |
| 荣耀 | 形容词（光荣） | 手机、HONOR、MagicOS | 无上、祖国、家族（作为修饰语） |
| 美的 | 形容词（美丽） | 空调、电器、冰箱、家电、洗衣机 | 美好、美丽（作为形容词修饰其他名词） |
| 联想 | 心理活动 | 电脑、笔记本、ThinkPad、拯救者、IdeaPad | 引起、让人、产生（作为动词/普通名词） |

### 6.2 消歧流程

```
输入: 文本片段 + 候选品牌词
  │
  ▼
Step 1: 歧义检查
  └─ 品牌词在歧义词表中吗？
      ├─ 否 → 直接按品牌处理，正常识别
      └─ 是 → Step 2

Step 2: 上下文搜索（前后15个字范围）
  └─ 搜索品类词/商品词
      ├─ 找到电子/数码/家电相关词 → 判定为品牌
      ├─ 找到食品/农产品相关词 → 判定为普通词（不是品牌）
      ├─ 找到其他品类词 → 按品类词判断合理性
      └─ 未找到品类词 → Step 3

Step 3: 上下文搜索（扩大范围）
  └─ 搜索店铺名、价格、平台等间接线索
      ├─ 有数码/电器相关线索 → 倾向品牌，降低置信度
      ├─ 有食品相关线索 → 倾向普通词
      └─ 完全无线索 → 标记 need_brand_review=true
```

### 6.3 跨品类品牌消歧

当品牌跨多个品类且只有品牌没有商品词时：

```
输入: "买了个小米，有问题"
  │
  ▼
品牌识别: 小米 → brand_id=BRAND_XIAOMI, confidence=0.90
  │
  ▼
品类判断: 只有品牌，无商品词
  │ 小米是跨品类品牌（手机/电视/家电/智能设备）
  │
  ▼
处理: 不推断品类 → 输出多个候选品类
  │ primary_category = null
  │ candidate_categories = [
  │   {l1:"数码电子", l2:"手机通讯", confidence: 0.35},
  │   {l1:"家用电器", l2:"大家电", confidence: 0.25},
  │   ...
  │ ]
  │ need_human_review = true
  │ human_review_reason = "HR-5: 品牌跨≥3个品类且上下文无法消歧"
```

## 7. 品牌词典维护流程

### 7.1 阶段一：种子词典（手动构建）

1. 梳理常见消费品牌 ≥20 个（优先高频投诉品牌）
2. 为每个品牌录入别名、产品线、常见品类
3. 标记歧义品牌和消歧规则
4. 输出 `stage1_brand_mapping_seed.json`

### 7.2 阶段二：从投诉数据中挖掘

1. LLM 从 24,713 条投诉标题中批量抽取品牌词
2. 按频次排序，取 TOP 200 品牌词
3. 人工审核：去除非品牌词、合并同品牌变体
4. 对每个新品牌，通过 LLM + 人工补充别名、产品线、常见品类
5. 品牌词典扩展到 ≥200 个品牌

### 7.3 阶段三：持续更新

1. 每周从人工复核的投诉中发现新品牌候选
2. 每月统计品牌投诉分布，发现新兴品牌
3. 季度 Review 品牌-品类映射的准确性
4. 品牌更名/合并/拆分时同步更新词典

### 7.4 从投诉数据挖掘新品牌候选的方法

1. **高频未知实体：** 实体抽取中出现的高频但未在品牌词典中的BRAND类型实体
2. **品牌词共现分析：** 与已知品牌词共现但本身不在词典中的词（可能是子品牌或新品牌）
3. **店铺名品牌提取：** 从"XX旗舰店""XX专卖店"中提取品牌名
4. **型号-品牌关联：** 新的型号/系列名与已知品类词共现
5. **标注反馈：** 标注员在试标中标记的未知品牌

## 9. 品牌识别流程（关键词命中≠最终品牌）

### 9.1 核心原则

**品牌识别不是简单关键词分类。** 关键词命中品牌词典只是候选生成（candidate generation），不是最终识别结果（final result）。

**识别流程：**

```
投诉文本
    │
    ▼
┌─────────────────────────────────┐
│ Step 1: 候选生成（Candidate）     │
│ - 品牌词典别名匹配               │

```

### 9.2 关键规则

1. **关键词命中只是 candidate，不是 final。** 品牌词典匹配到的所有品牌提及都是候选，必须经过消歧和置信度评估。
2. **歧义词必须做上下文消歧。** `苹果`、`小米`、`荣耀`、`美的`、`联想` 等词不得直接输出为品牌，必须检查上下文。
3. **短别名不能单独高置信度触发。** `MI`、`Mac`、`Mate`、`HP`、`OP` 等短词（≤3字符）必须与商品词或更长品牌词共现。
4. **产品线命中比短别名更可靠。** `iPhone`、`iPad`、`AirPods`、`MacBook` 比 `Mac` 更可靠；`MateBook`、`MatePad` 比 `Mate` 更可靠。
5. **型号命中需通过型号-品牌-品类映射确认。** `RTX 5090` → NVIDIA/英伟达 → 显卡。
6. **平台名不是品牌。** 淘宝、京东、拼多多、抖音、得物 等是电商平台，不是商品品牌。
7. **商家名中的"官方客服""售后客服"不是品牌。** 需要从商家名中提取真正的品牌名。
8. **如果只有品牌没有商品词且品牌跨多品类，必须人工复核。**

## 10. 品牌置信度规则

### 10.1 基础置信度表

品牌识别置信度分为两级：
- **candidate_match_confidence（候选命中置信度）：** 词典/别名/产品线命中的初始置信度
- **final_brand_confidence（最终品牌置信度）：** 经过上下文消歧、非品牌排除后的最终置信度

`brand_confidence` 字段输出的必须是 `final_brand_confidence`，不是候选命中置信度。

| 命中类型 | 示例 | 候选置信度 | 最终置信度 | 说明 |
|---------|------|-----------|-----------|------|
| 官方品牌名精确命中 + 品牌上下文一致 | "华为手机"、"Nike鞋" | 0.90-0.98 | 0.90-0.98 | 品牌上下文关键词共现，可自动识别 |
| 产品线精确命中 | iPhone、iPad、AirPods、MacBook | 0.88 | 0.85-0.95 | 产品线名具有强品牌指向性 |
| 型号/系列精确命中 | RTX 5090、Mate 60、Galaxy S24 | 0.88 | 0.80-0.92 | 需通过型号→品牌→品类映射确认 |
| 中文/英文别名命中 + 品牌上下文 | "耐克球鞋"、"Apple Watch" | 0.88-0.93 | 0.80-0.95 | 有品牌上下文关键词支撑 |
| 歧义词命中 + 品牌上下文明确 | "买了一台苹果手机" | 0.55 | 0.80-0.95 | 消歧通过后可以提升到高置信度 |
| 歧义词命中 + 无非品牌上下文 | "苹果有问题" | 0.55 | 0.40-0.55 | 无法确认是品牌还是普通词 |
| 歧义词命中 + 非品牌上下文 | "昭通苹果尺寸不符" | 0.55 | **0** | 明确非品牌语境（水果/粮食/形容词），强制判定为非品牌 |
| 短别名命中 + 品牌上下文 | "Mac电脑"、"Mate手机" | 0.65 | 0.70-0.85 | 有品牌上下文可适度提升 |
| 短别名命中 + 无上下文 | "买了个Mac" | 0.65 | 0.50-0.60 | 无品牌上下文，必须复核 |
| 店铺名命中 | XX旗舰店、XX专卖店 | 0.60-0.85 | 0.60-0.85 | 辅助证据，不能单独决定 |
| 模糊拼写/错别字命中 | Nvida、Samsng、mete | 0.50-0.75 | 0.50-0.70 | 低置信度，建议复核 |

### 10.2 置信度调节规则

| 调节条件 | 调节方向 | 调节幅度 | 说明 |
|---------|---------|---------|------|
| 品牌与商品品类一致（如Nike+鞋） | 上调 | +0.05~0.10 | 品牌-品类一致性验证通过 |
| 品牌与商品品类冲突（如Apple+食品） | **强制置零** | 置信度=0 | **歧义词命中 + 非品牌上下文 → 判定为非品牌** |
| 多证据交叉验证（标题+内容+商家都指向同一品牌） | 上调 | +0.05~0.10 | 多渠道证据一致性高 |
| 只有品牌没有商品词 | 上限锁定 | 置信度不得超过 0.75 | 即使单一品类品牌也不例外 |
| 跨品类品牌 + 只有品牌词 | 上限锁定 + 强制复核 | 置信度不得超过 0.60 | 必须 need_brand_review=true |
| 短别名 + 无上下文商品词 | 强制复核 | 置信度 ≤0.60 | 必须 need_brand_review=true |
| 歧义词 + 无法消歧（非品牌/品牌上下文都无） | 强制复核 | 置信度 ≤0.55 | 必须 need_brand_review=true |
| 歧义词 + 明确非品牌上下文 | **强制置零** | 置信度=0 | **不识别为品牌**，main_brand=null |
| 模糊拼写/错别字 + 疑似型号 | 强制复核 | 置信度 0.60-0.75 | 如"mete80promax"→疑似Mate系列 |
| 仅在商家名出现（标题/内容未出现） | 下调 | -0.10~-0.20 | 商家名品牌不等于投诉商品品牌 |
| 仅标题/内容出现，商家名未出现 | 不调 | ±0 | 标题/内容是最可靠的品牌来源 |

### 10.3 歧义词非品牌上下文判定规则（关键）

以下语境中，歧义词**不是品牌，不得输出品牌识别结果**：

| 歧义词 | 非品牌语境关键词 | 典型非品牌示例 | 判定 |
|--------|---------------|-------------|------|
| 苹果 | 昭通、云南、阿克苏、烟台、水果、新鲜、尺寸、果径、mm、斤、箱、甜、酸、生鲜、坏果、果农、产地、冰糖心、红富士 | "昭通苹果尺寸不符"、"云南苹果新鲜" | NOT_BRAND, confidence=0 |
| 小米 | 粮食、煮粥、一斤、杂粮、黄小米、小米粥、谷物、五谷、粗粮 | "买了两斤小米煮粥" | NOT_BRAND, confidence=0 |
| 美的 | 美好、美丽、风景、生活、时光、景色、精美、优美 | "风景很美的"、"美好的生活" | NOT_BRAND, confidence=0 |
| 联想 | 引起、让人、产生、想象、丰富、带来、激发、勾起 | "引起丰富的联想" | NOT_BRAND, confidence=0 |
| 荣耀 | 无上、祖国、家族、军人、勋章、光荣、荣誉感、至高 | "无上荣耀"、"祖国的荣耀" | NOT_BRAND, confidence=0 |

**判定流程：**
1. 检查文本中是否包含 `non_brand_context_keywords`
2. 如果包含且不包含 `brand_context_keywords` → 判定为 NOT_BRAND
3. 如果同时包含品牌和非品牌关键词 → 以更多、更具体的上下文为准
4. 如果都不包含 → 标记为无法消歧，need_brand_review=true

### 10.4 候选去重规则

### 10.4 候选去重规则

品牌候选经常出现重复（同一品牌别名在标题和正文中多次出现）。去重规则：

1. **去重键：** `raw_text（小写） + brand_id + source_field`
2. **保留规则：** 同品牌同字段首次出现保留，后续重复合并
3. **输出标记：** 如果某品牌提及在文本中出现多次，在 `brand_evidence_text` 中只记录一次，在 `notes` 中标注"重复提及已合并"
4. **置信度不叠加：** 重复匹配不增加置信度（同一字段内多次出现不提升置信度）
5. **跨字段证据：** 标题+内容+商家同时命中同一品牌 → 作为交叉验证提升置信度（已在置信度调节规则中处理）

### 10.5 最终置信度分类

| 最终置信度 | 分类 | 处理 |
|-----------|------|------|
| ≥0.85 | 高置信度 | 可自动输出品牌，抽样校验 |
| 0.70-0.84 | 中置信度 | 可自动输出品牌，建议标注员确认 |
| 0.50-0.69 | 低置信度 | need_brand_review=true，建议人工复核 |
| <0.50 | 不可靠 | need_brand_review=true，必须人工复核 |
| 无法判断 | — | main_brand=null，detected_brands=[] |

## 11. 产品线映射强化规则

### 11.1 产品线/型号不是普通关键词

产品线和型号是品牌归一化的重要依据，其可靠性和指向性远高于短别名和歧义词。

**产品线→品牌→品类映射示例：**

| 产品线/型号 | → 品牌 | → 候选品类 | 说明 |
|-----------|--------|-----------|------|
| iPhone | Apple/苹果 | 数码电子 > 手机通讯 | 强指向性，iPhone仅有Apple一个品牌 |
| iPad | Apple/苹果 | 数码电子 > 电脑整机与硬件 | 平板电脑 |
| MacBook | Apple/苹果 | 数码电子 > 电脑整机与硬件 | 笔记本。比简称"Mac"更可靠 |
| AirPods | Apple/苹果 | 数码电子 > 影音娱乐与游戏设备 | 无线耳机 |
| Apple Watch | Apple/苹果 | 数码电子 > 智能穿戴与数码配件 | 名称本身含"Apple" |
| Mate 60 | 华为 | 数码电子 > 手机通讯 | 需注意Mate也可能指MateBook/MatePad |
| MateBook | 华为 | 数码电子 > 电脑整机与硬件 | 比简称"Mate"更可靠 |
| MatePad | 华为 | 数码电子 > 电脑整机与硬件 | 平板 |
| Pura 70 | 华为 | 数码电子 > 手机通讯 | 原P系列 |
| Redmi/红米 | 小米/小米集团（子品牌） | 数码电子 > 手机通讯 | 子品牌归属明确 |
| 米家 | 小米/小米集团（子品牌） | 家用电器/智能家居 | 智能家居品牌，品类较分散 |
| Galaxy S24 | Samsung/三星 | 数码电子 > 手机通讯 | |

| RTX 5090 | NVIDIA/英伟达 | 数码电子 > 电脑整机与硬件（显卡） | 型号→品牌映射强 |
| Ryzen 7 | AMD | 数码电子 > 电脑整机与硬件（CPU） | |

### 11.2 短别名的特殊性

短别名（≤3字符或通用性极强的词）不可单独用于品牌判定：

| 短别名 | 对应品牌 | 问题 | 处理规则 |
|--------|---------|------|---------|
| MI | 小米 | 太短，可能是拼音mi或其他缩写 | 必须与"手机""电视""Redmi""Xiaomi"等词共现 |
| Mac | Apple | 单独出现可能指Mac电脑，但太短 | 优先找"MacBook""iMac""Mac mini"等更长产品线名 |
| Mate | 华为 | 单独出现可能指Mate手机，但"Mate"也是英文单词 | 优先找"Mate 60""MateBook""MatePad"等更具体型号 |
| HP | 惠普 | 太短，可能被误匹配 | 必须有电脑/笔记本/打印机等上下文词 |
| OPPO | OPPO | 虽然基本无歧义，但在非常用语境中需验证 | 检查上下文是否为消费电子 |

**短别名处理规则：**
1. 如果短别名与更长的产品线/型号名共现（如"Mate 60"中的"Mate"），以更长名称为准，置信度正常。
2. 如果短别名单独出现且无商品词 → `need_brand_review=true`。
3. 如果短别名单独出现但有商品词 → 置信度上限 +0.10（如0.50→0.60），仍建议人工确认。
