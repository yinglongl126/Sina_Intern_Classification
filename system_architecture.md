# 系统架构设计（第二版补充）

## 版本说明

本文档在 v1 架构基础上，强化了**实体抽取作为第一核心模块**的定位。

v1原有架构图、模块设计、数据库设计、API设计等内容保留。本文件仅新增/修改实体抽取在架构中的前置位置。

---

## [修改] 整体架构 — 实体抽取前置

```
                            ┌──────────────────────────────────────────────┐
                            │              线上推理服务                       │
                            │                                              │
  新投诉数据 ──────────────> │  ┌──────────┐   ┌──────────────┐   ┌──────┐ │
  (API/批量)               │  │ 文本预处理 │──>│ 实体抽取 ★   │──>│ 品类 │ │
                            │  └──────────┘   │ (第一核心模块) │   │ 分类 │ │
                            │                  └──────────────┘   └──────┘ │
                            │                         │               │    │
                            │                         ▼               ▼    │
                            │              ┌─────────────────────────────┐  │
                            │              │   置信度判断 + 分流决策      │  │
                            │              └─────────────────────────────┘  │
                            └──────────────────────┬───────────────────────┘
                                                   │
                    ┌──────────────────────────────┼──────────────────────┐
                    │                              │                      │
                    v                              v                      v
            ┌──────────────┐              ┌──────────────┐      ┌──────────────┐
            │  自动分流     │              │  人工复核     │      │  争议升级     │
            │ (高置信度)    │              │ (低/无实体)   │      │ (新品类/合规) │
            └──────────────┘              └──────────────┘      └──────────────┘
```

**关键变化：** 实体抽取从原来的"2.2 实体抽取模块（附属）" 升级为 "第一核心模块"，位于文本预处理之后、品类分类之前。详见 `entity_extraction_design.md`。
