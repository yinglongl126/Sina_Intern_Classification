# 商品/服务品类自动分类与分流系统 — 项目概述（第二版补充）

## 版本说明

本文档在 v1 的基础上补充以下内容（v1 原文保留在下方）。新增内容聚焦三个问题：
1. 明确主维度 vs 辅助维度的边界
2. 明确当前阶段优先 L1/L2，不直接追求完整 L3
3. 强调实体抽取前置

## [新增] 0. 维度定位声明

### 本项目的主维度（必须做的）

**商品/服务品类分类** — 回答"投诉涉及的**是什么**商品/服务"

- 这是本项目的核心目标
- 分类结果决定分流方向
- 是统计、监管、治理的基础维度

### 本项目的辅助维度（可以做，但不是核心）

**投诉问题类型** — 回答"消费者为什么投诉"（质量问题、退款纠纷、虚假宣传等）
**诉求类型** — 回答"消费者想要什么"（退款、赔偿、换货等）
**风险等级** — 回答"这条投诉有多紧急"

**辅助维度不作为分流的主要依据，不作为品类树的内容。**

### 当前阶段目标

**阶段一核心目标：** L1+L2 品类自动分类，支撑初步分流
**阶段一不追求：** 完整的 L3 分类体系、全自动分流、高准确率模型

## [保留] 1. 项目要解决的问题

**核心问题：** 当一条新的消费者投诉数据进入系统时，运营人员或系统需要快速判断"这条投诉涉及的是什么商品或服务"，并将其分流到对应的处理团队。

**现状痛点（基于对实际数据的分析）：**

当前数据集（24713条电商投诉）的已有分类字段仅有"购物平台 > 网上商城"，这是对**投诉渠道**的分类，而不是对**投诉涉及的商品/服务品类**的分类。

这意味着**运营人员无法按商品品类统计投诉量、无法自动分流、无法识别高风险品类**。

## [保留] 2. 与"投诉问题分类"的区别

| 维度 | 投诉问题分类 | 商品/服务品类分类（本项目） |
|------|-------------|--------------------------|
| **分类对象** | 消费者投诉的原因/问题类型 | 投诉涉及的商品或服务属于哪个品类 |
| **示例：显卡温度高黑屏** | 问题类型 = "质量问题" | 品类 = "数码电子 > 电脑整机与硬件" |
| **示例：会员无法使用** | 问题类型 = "服务故障" | 品类 = "虚拟商品 > 会员与订阅服务" |
| **用途** | 分析投诉原因 | 分流处理、品类风险监控、行业监管统计 |

**红线：** "退款"、"质量差"、"虚假宣传"、"物流慢"、"客服态度差"等是投诉问题，不是商品品类。

## [新增] 阶段一范围声明

### 阶段一做什么

1. 从投诉文本中抽取商品/服务实体
2. 归入 L1 → L2 品类（20个业务L1 + 1个兜底L1，114个L2）
3. 判断服务对象类型
4. 给出置信度
5. 标记是否需要人工复核
6. 推荐分流方向

### 阶段一不做什么

1. 不追求 L3 细分（仅对6-8个高频品类试点L3）
2. 不上线自动分流（先离线验证分类效果）
3. 不训练小模型（先用LLM积累标注数据）
4. 不追求全品类覆盖（允许≤30%进入人工复核）

### 阶段一验收标准

- L1自动分类准确率 ≥ 80%
- L2自动分类准确率 ≥ 70%
- 实体抽取成功率 ≥ 85%
- 标注数据积累 ≥ 3000条
- 标注一致性 Kappa ≥ 0.70
