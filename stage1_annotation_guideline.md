# 阶段一人工标注指南

## 0. 文档说明

本文档供标注员在第一阶段标注工作中使用。

**标注目标：** 阅读投诉文本，标注该投诉涉及的商品/服务品类（L1+L2层级）和商品/服务实体。

**标注范围：** 阶段一只标注 L1 和 L2，不要求标注 L3（L3 在后续阶段才建设）。

---

## 1. 核心规则（标注员必读）

### 规则1：你标注的是"投诉对象"，不是"投诉原因"

这是最重要的规则。请在标注每条投诉前先问自己：

> "这条投诉在说什么商品或服务？"

**不要问自己：**
> "消费者为什么投诉？" ← 这是另一个问题

**正确示例：**

| 投诉文本 | ✅ 应该标注的品类 | ❌ 不应该标注 |
|---------|----------------|------------|
| "显卡温度过高黑屏" | 数码电子 > 电脑整机与硬件 | 质量问题 |
| "羽绒服掉毛严重" | 服饰鞋包 > 女装/男装 | 商品质量问题 |
| "外卖送了一个小时还没到" | 本地生活服务 > 餐饮外卖 | 配送慢 |
| "账号无故被封禁" | 平台服务 > 账号服务 | 客服不处理 |

### 规则2：先找商品/服务实体，再定品类

阅读顺序：
1. **先读标题** — 标题通常直接包含商品/服务词
2. **再读内容首段** — 正文开头往往补充商品细节
3. **扫一眼商家名** — 看是否包含品类线索
4. **最后决定品类** — 基于找到的商品/服务实体选择L1→L2

### 规则3：找不到商品/服务实体时，不要强行分类

如果标题和正文中完全没有提及具体商品或服务，只有"退款"、"投诉"、"太差了"等词，请标注为：

- L1: 其他/待人工复核 (98)
- L2: 暂无法归类 (98-01)

**不要在"退款"后面加一个猜测的品类。**

---

## 2. 标注流程

```
Step 1: 阅读标题
  └─ 找到商品/服务词了吗？
      ├─ 是 → 记录实体词 → Step 2
      └─ 否 → 阅读内容首段 → 找到了吗？
               ├─ 是 → 记录实体词 → Step 2
               └─ 否 → 阅读全文+商家名 → 找到了吗？
                        ├─ 是 → 记录实体词 → Step 2
                        └─ 否 → 标注为"暂无法归类" (98-01)
Step 2: 判断实体类型
  └─ 实体是实物商品？ → PHYSICAL
  └─ 实体是虚拟商品/数字服务？ → VIRTUAL
  └─ 实体是线下服务？ → LOCAL
  └─ 实体是平台功能/账号？ → PLATFORM
Step 3: 选择 L1 → L2
  └─ 在品类表中找到最匹配的分类
Step 4: 检查是否有第二商品/服务
  └─ 有 → 标注为辅品类
  └─ 无 → 完成
Step 5: 判断是否需要人工复核标记
  └─ 不确定？多商品冲突？信息不足？ → 标记need_review=true
```

---

## 3. 多商品投诉处理

### 3.1 如何确定主商品

| 优先级 | 规则 |
|--------|------|
| 1 | 标题中最先出现的商品/服务 |
| 2 | 金额最接近的商品 |
| 3 | 投诉核心聚焦的商品（看消费者最关心哪个） |

### 3.2 标注格式

```
主品类: 数码电子 > 电脑整机与硬件（显卡）
辅品类: 数码电子 > 电脑外设与网络设备（显示器）
```

**注意：** 辅品类不是必填项。只在确实有第二个明确商品时才标。

---

## 4. 什么情况标记"人工复核"

以下情况在标注结果中标记 `need_review=true`：

1. **找不到任何商品/服务实体** — 标题和正文只有"退款"、"投诉"等词
2. **品牌跨多个品类且无法确定** — 如"小米"可能是手机、电视、扫地机
3. **投诉涉及的商品不在现有L2中** — 疑似新品类
4. **信息严重不足** — 投诉只有几个字，如"质量差"、"假货"
5. **多商品同等重要，无法区分主次**

## 5. 边界案例处理

### 5.1 "退款"投诉

| 投诉 | 处理 |
|------|------|
| "买的手机要退款" | 品类=手机（有商品实体） |
| "要求退款" | 标注为"暂无法归类"（无商品实体） |
| "退款一直不到账" | 标注为"暂无法归类"（无商品实体） |

### 5.2 直播间相关

| 投诉 | 处理 |
|------|------|
| "在直播间买的手机是假的" | 品类=手机（商品是手机，直播间是渠道） |
| "我的直播间被平台封了" | 品类=平台服务>直播与内容服务（投诉对象是直播间本身） |

### 5.3 物流/快递相关

| 投诉 | 处理 |
|------|------|
| "快递把我的手机弄丢了" | 品类=手机（投诉对象是手机，快递是中间环节） |
| "快递公司把我的包裹弄丢了" | 品类=平台服务>物流配送服务（投诉对象是快递服务本身） |
| "物流信息一直不更新" | 品类=平台服务>物流配送服务 |

### 5.4 运费险

| 投诉 | 处理 |
|------|------|
| "运费险不赔付" | 品类=金融支付>保险产品（投诉对象是保险产品） |
| "退货了要退运费" | 如有商品信息→商品品类；否则→暂无法归类 |

### 5.5 平台会员/优惠券

| 投诉 | 处理 |
|------|------|
| "拼多多会员不能用了" | 品类=虚拟商品>会员与订阅服务 |
| "优惠券过期了" | 品类=虚拟商品>数字卡券与充值 |
| "买手机送的优惠券用不了" | 品类=手机（主），辅=虚拟商品>数字卡券 |

### 5.6 外卖/餐饮

| 投诉 | 处理 |
|------|------|
| "外卖送了一个小时还没到" | 品类=本地生活服务>餐饮外卖 |
| "外卖里的菜是坏的" | 品类=本地生活服务>餐饮外卖 |
| "在拼多多买了水果坏了" | 品类=食品生鲜>生鲜水果（这是电商购物，不是外卖） |

## 9. 常见错误

| 错误 | 正确做法 |
|------|---------|
| 把"退款失败"当作品类 | 退款不是品类。找到被退款的商品，标注该商品的品类 |
| 把"客服态度差"当作品类 | 客服态度不是品类。找到投诉中提及的具体商品或服务 |
| 把"虚假宣传"当作品类 | 虚假宣传不是品类。找到被虚假宣传的商品 |
| 把平台名当作品类 | "拼多多"不是品类。找到在拼多多上购买的具体商品 |
| 看到"苹果"就标食品生鲜 | 先判断上下文：是水果还是iPhone |
| 看到"小米"就标数码电子 | "小米"也可能指小米电视(家电)、小米空气炸锅(家电) |
| 看到"三包"就当成箱包 | "三包"是保修/包修/包换的消费者权益术语，不是商品实体。要从内容中找真实的商品词（如"衣服"）。 |
| 看到"苹果"、"apple"就归食品生鲜 | 先判断上下文：有数码线索（手机/电脑/iOS）→数码电子；有水果线索（新鲜/斤/甜/产地）→食品生鲜；完全无线索→标记人工复核，不能默认归水果。 |
| 看到英文大写词就当作品牌 | 需要结合上下文：CARHARTT/Carhartt→品牌；三包/七天无理由等→消费者术语。 |

## 10. Apple/苹果歧义规则（新增）

### 规则说明

`苹果` 和 `apple` 同时具有水果和 Apple 品牌两种含义，必须结合上下文判断。

### 判定流程

1. **数码电子上下文** → Apple 品牌
   - 关键词：手机、电脑、iPhone、iPad、iOS、MacBook、AirPods、平板、笔记本、数码、app store
   - 处理：品牌=Apple/苹果，品类=根据具体商品词确定
   - 示例："苹果手机退款" → 品类=手机通讯

2. **食品生鲜上下文** → 水果
   - 关键词：新鲜、昭通、云南、产地、果径、mm、斤、箱、甜、酸、水果、生鲜、坏果、果农
   - 处理：品牌=N/A（非品牌），品类=食品生鲜>生鲜水果
   - 示例："昭通苹果尺寸不符" → 品类=生鲜水果

3. **完全无线索** → 标记人工复核
   - 标题只有"苹果有问题"、"苹果降价了"、"苹果商品"等
   - 处理：归入 98-01，need_human_review=TRUE
   - human_review_reason：HR-APPLE-AMBIGUITY
   - 示例："淘宝百亿补贴苹果商品下单难" → 人工复核

### 重要

**不能因为出现"苹果"就默认归入食品生鲜。** 必须通过上下文消歧。无上下文时标记人工复核。

## 11. 服装性别判断规则（新增）

### 规则说明

`stage1_taxonomy.json` 中服饰鞋包的 L2 包含 `女装(02-01)`、`男装(02-02)`、`童装与亲子装(02-03)`，但没有中性服装 L2。

### 判定规则

| 文本线索 | 归入 L2 | 置信度 |
|---------|--------|--------|
| 明确女性线索：女款、女士、女装、W、裙、女性尺码 | 02-01 女装 | 正常 |
| 明确男性线索：男款、男士、男装、男码、工装夹克、男外套 | 02-02 男装 | 正常 |
| 明确童装线索：童装、儿童、婴儿服、校服 | 02-03 童装与亲子装 | 正常 |
| 无性别线索：羽绒服/外套/衣服/裤子等中性词 | **标记人工复核** | 降低 |

### 无性别线索时的处理

- 优先寻找隐藏线索：品牌定位（如"骆驼户外"偏中性）、款式描述
- 如果确实无任何性别线索：可选择最可能的 L2（通常 02-01 女装作为默认），但必须：
  - `need_human_review=TRUE`
  - `human_review_reason=HR-GENDER-UNKNOWN`
  - `category_confidence ≤ 0.65`
- 示例："得物购买Salomon羽绒服退货退款" → 候选 02-01 但需复核

## 14. "三包"等消费者术语误匹配警示（新增）

### 规则说明

消费者权益相关术语不应被抽取为商品实体。

### 常见误匹配

| 术语 | 含义 | 正确处理 |
|------|------|---------|
| 三包 | 保修/包修/包换 | 不是商品实体，从内容中找真实商品词 |
| 七天无理由 | 退货权利 | 不是商品实体 |
| 假一赔三 | 赔偿诉求 | 不是商品实体 |
| 价保/保价 | 价格保护 | 不是商品实体 |
| 单笔/多笔（订单量词） | 订单计数单位 | 不是商品实体。"单笔订单"中的"笔"≠文具笔 |

### 示例

- 标题"衣服三包问题" → 商品实体="衣服"（非"包"）
- "单笔订单赔付" → 无商品实体，"笔"为量词（非水性笔/文具）

## 14-附. "店铺购买 + 商品词"主实体判定规则（v2.3 人工抽检新增）

### 规则说明

当投诉文本同时出现"在某店铺/平台购买"和"明确商品词"时，商品词优先作为主实体。"店铺"只是购买渠道/商家信息，不应覆盖商品实体。

### 判定规则

| 文本模式 | 主实体 | 说明 |
|---------|--------|------|
| "在XX店铺购买YY商品，YY有问题" | YY商品 | 商品是投诉对象 |
| "XX店铺不发货" | 店铺 | 投诉对象是店铺服务 |
| "XX店铺虚假宣传/欺诈" | 看具体商品 | 先找商品词，有则商品优先 |
| "YY平台客服不作为" | 无商品词→店铺/平台服务 | 纯服务投诉 |

### 示例

- "在淘宝网酷姐姐星杯店铺购买星巴克杯子，杯子漏水做工粗糙" → 主实体=杯子（非店铺）
- "拼多多XX店铺一直不发货" → 主实体=店铺（无商品词）→ 平台服务
- "京东自营买了手机不发货" → 主实体=手机（商品词明确，店铺为渠道）

## 15. 中文成语/固定表达误匹配警示（v2.3 新增）

### 规则说明

中文成语和固定表达中包含的动物/物品字，不应被抽取为商品实体。

### 常见误匹配

| 表达 | 含义 | 正确处理 |
|------|------|---------|
| 挂羊头卖狗肉 | 虚假宣传（以好充次） | 不是商品实体。"羊"和"狗"均不能作为宠物/食品实体抽取。从内容中找真实商品词 |
| 猫腻 | 暗箱操作/隐藏问题 | 不是商品实体。"猫"不能作为宠物抽取 |
| 踢皮球 | 互相推诿 | 不是商品实体 |
| 割韭菜 | 欺瞒消费者 | 不是商品实体 |

### 判定流程

1. 如果动物字出现在成语或固定表达中 → 不作为商品实体
2. 如果动物字出现在明确的宠物购买/饲养语境中（"购买一只狗"、"猫猫真菌感染"） → 属于商品实体
3. 如果动物字出现在食品语境中（"牛肉干"、"鸡肉"） → 属于食品实体

### 示例

- "商家虚假宣传，挂羊头卖狗肉" → 商品实体=衣服（从内容提取），非狗/羊
- "租机猫腻多" → 商品实体=手机（从内容提取），非猫
- "平台踢皮球不处理" → 无商品实体 → 兜底

## 16. 活体宠物 vs 宠物食品边界（v2.3 新增）

### 规则说明

`stage1_taxonomy.json` 中宠物用品包含 `宠物食品(13-01)`、`宠物用品与清洁(13-02)`、`宠物医疗与保健(13-03)`、`宠物用品其他(13-99)`，但没有专用的**活体宠物** L2。

### 判定规则

| 投诉对象 | 归入 L2 | 说明 |
|---------|--------|------|
| 猫粮、狗粮、宠物零食、营养品、罐头 | 13-01 宠物食品 | 食品类 |
| 猫砂、狗绳、宠物窝、玩具 | 13-02 宠物用品与清洁 | 用品类 |
| 驱虫药、宠物药品 | 13-03 宠物医疗与保健 | 医疗类 |
| **活体猫、活体狗、活体鹦鹉、活体鱼等** | **13-99 宠物用品其他** + **人工复核** | 活体，非"用品" |

### 活体宠物处理

- 归入 13-99 宠物用品其他（最近似 L2）
- 必须标记 `need_human_review=TRUE`
- `human_review_reason=HR-LIVE-PET-BOUNDARY`
- `category_confidence ≤ 0.70`
- 示例：
  - "购买一只宠物狗，收到瑕疵狗" → 13-99 + 复核
  - "购买蓝金渐层猫猫，真菌感染" → 13-99 + 复核
  - "购买玄凤鹦鹉，脚趾残缺" → 13-99 + 复核
