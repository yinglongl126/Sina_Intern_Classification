# 实施计划（第二版 — 现实主义路线）

## 0. 版本说明

第一版计划过于激进：2周内从小样本到训练600类L3模型。
第二版改为**先验证分类体系、再建模型、最后上线**的务实路线。

**核心原则：**
- 先验证"能不能分对"，再追求"能不能自动分"
- 先做L1+L2，再逐步细化L3
- 先确保人工标注一致性，再训练模型
- 每个阶段有明确的验收标准，达标后再推进

---

## 1. 总体路线图

```
Phase 1 (3-4周)          Phase 2 (3-4周)          Phase 3 (4周+)         Phase 4 (持续)
──────────────────────────────────────────────────────────────────────────────────
[验证分类体系]  ──────> [建设自动分类] ──────> [规模化上线] ──────> [持续优化]

实体抽取+LLM分类         商品词典+轻量模型         全量分类+分流上线       模型迭代+品类维护
人工标注+一致性校验       自动分流试点             运营团队对接            新品类发现
标注数据积累             模型评估+调优            人工复核流程            反馈闭环
L1/L2体系验证            L3试点                  效果评估                成本优化
```

## 2. Phase 1：分类体系验证与标注数据积累（第1-4周）

### 目标
验证品类体系可用、建立标注标准、积累≥3000条标注数据

### 第1周：准备与试点

| 任务 | 说明 | 产出 | 工时 |
|------|------|------|------|
| 1.1 确认阶段一品类体系 | 基于stage1_taxonomy.json review | 确认版L1/L2品类表 | 0.5天 |
| 1.2 设计LLM Prompt | 含实体抽取+品类分类的联合Prompt | Prompt模板(含10-15个few-shot示例) | 1天 |
| 1.3 试点分类100条 | 随机抽取100条，人工标注+LLM分类对比 | 试点评估：L1/L2准确率、实体抽取效果 | 1天 |
| 1.4 标注指南编写 | 标注规则、示例、边界case处理 | stage1_annotation_guideline.md | 1天 |
| 1.5 标注工具准备 | Excel模板或简易标注界面 | 标注模板 | 0.5天 |

**第1周验收：**
- ✅ Prompt在100条试点数据上L1准确率≥75%
- ✅ 实体抽取成功率≥80%
- ✅ 标注指南完成

### 第2周：第一轮标注

| 任务 | 说明 | 产出 | 工时 |
|------|------|------|------|
| 2.1 标注培训 | 2名标注员培训，试标20条讨论 | 标注一致性对齐 | 0.5天 |
| 2.2 双人标注200条（一致性检查） | 同一批数据，两人独立标注 | 标注一致性报告 | 1天 |
| 2.3 如果一致性<0.70：修订标注指南，再标100条 | 直到一致性达标 | 修订后的标注指南 | 1天 |
| 2.4 LLM批量分类1500条 | 覆盖全部L1品类 | 1500条LLM分类结果 | 1天 |

**第2周验收：**
- ✅ 标注一致性 Kappa ≥ 0.70
- ✅ 1500条LLM分类结果产出

### 第3周：质量检查与标注扩量

| 任务 | 说明 | 产出 | 工时 |
|------|------|------|------|
| 3.1 抽样校验300条（从1500条中） | 按品类分层抽样，覆盖所有L1 | 校验准确率报告 | 2天 |
| 3.2 问题分析和规则优化 | 分析分类错误模式，优化Prompt | 优化后的Prompt | 1天 |
| 3.3 LLM扩量分类2000条 | 用优化后的Prompt | 2000条LLM分类结果 | 1天 |
| 3.4 商品词典初版建设 | 从已分类数据中提取商品词→L2映射 | ≥300个商品词 | 1天 |
| 3.5 品牌映射库初版 | 从数据中提取品牌→默认品类映射 | ≥100个品牌 | 0.5天 |

**第3周验收：**
- ✅ 累计≥3000条LLM分类+≥500条人工校验标注
- ✅ L1准确率≥80%（在人工校验subset上）
- ✅ 商品词典≥300条

### 第4周：Phase 1总结

| 任务 | 说明 | 产出 | 工时 |
|------|------|------|------|
| 4.1 分类体系问题总结 | 哪些L1/L2覆盖好、哪些缺失 | 品类体系优化建议 | 1天 |
| 4.2 标注数据整理 | 合并校验数据，生成训练集候选 | 训练集(≥3000条) | 0.5天 |
| 4.3 实体抽取效果评估 | 评估词典+LLM的实体抽取效果 | 实体抽取评估报告 | 0.5天 |
| 4.4 Phase 1总结报告 | 指标达成、问题清单、Phase 2建议 | Phase 1总结报告 | 0.5天 |
| 4.5 Phase 2 Go/No-Go决策 | 基于指标决定是否推进 | 决策文档 | 0.5天 |

**Phase 1 总验收标准：**
- ✅ 标注数据量 ≥ 3000条
- ✅ 人工校验量 ≥ 500条
- ✅ L1准确率（校验subset）≥ 80%
- ✅ L2准确率（校验subset）≥ 70%
- ✅ 标注一致性 Kappa ≥ 0.70
- ✅ 商品词典 ≥ 300条
- ✅ 品类体系问题清单完成
- ✅ **注意：Phase 1不要求模型训练、不要求L3分类、不要求上线**

## 3. Phase 2：自动分类系统建设（第5-8周）

### 前置条件（必须满足才能启动）
- Phase 1验收全部通过
- 标注数据≥3000条可用
- 标注一致性≥0.70

### 第5-6周：模型训练与评估

| 任务 | 说明 | 产出 |
|------|------|------|
| 5.1 训练集/验证集划分 | 80%/20%分层划分 | 训练/验证数据 |
| 5.2 MacBERT L1+L2分类模型训练 | 目标约135类输出（20业务L1 + 1兜底L1 + 114 L2） | 训练好的模型 |
| 5.3 模型评估 | L1/L2准确率、品类维度分析 | 评估报告 |
| 5.4 置信度阈值校准 | 找最优自动通过阈值 | 阈值配置 |
| 5.5 商品词典引擎开发 | 精确匹配+模糊匹配 | 规则引擎服务 |
| 5.6 分类API开发（单条+批量） | FastAPI服务 | API服务 |

**第6周验收：**
- ✅ 小模型L1准确率≥85%、L2准确率≥75%（验证集）
- ✅ 词典引擎覆盖率≥30%
- ✅ API服务可调用

### 第7-8周：系统集成与试点

| 任务 | 说明 | 产出 |
|------|------|------|
| 7.1 多层漏斗集成 | Layer 0(词典)+Layer 1(小模型)+Layer 2(LLM)+Layer 3(人工) | 完整Pipeline |
| 7.2 数据处理全量试点 | 对24713条全量数据做分类（离线） | 全量分类结果 |
| 7.3 效果评估 | 抽样200条评估端到端准确率 | 端到端评估报告 |
| 7.4 人工复核界面开发 | 简易Web标注界面 | 复核工具 |

**Phase 2 总验收标准：**
- ✅ 自动覆盖率 ≥ 60%（词典+小模型处理的比例）
- ✅ LLM兜底占比 ≤ 25%
- ✅ 人工复核占比 ≤ 20%
- ✅ 端到端L1准确率 ≥ 85%

## 7. 风险与应对（保留第一版内容，新增）

| 风险 | 新增应对 |
|------|---------|
| 标注一致性不达标 | 增加标注培训、细化指南、多做样本讨论 |
| LLM分类准确率不达标 | 优化Prompt、增加few-shot示例、缩小分类范围到L1 only |
| "其他"占比过高(>40%) | 检查是否为品类体系问题；考虑扩充L2 |
| 标注数据不足3000条 | 降低Phase 1目标到2000条；减少校验量 |
| 小模型效果差 | 先只做L1分类（21类），L2分类用LLM兜底 |
