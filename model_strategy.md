# 模型策略设计（第二版 — 现实主义路线）

## 0. 版本说明

第一版的问题：
- 假设冷启动后2周就能训练600类L3模型（不现实）
- L3准确率目标85%+（当前无标注数据，不可能）
- 自动覆盖率>85%（高估了初期模型能力）

第二版改为：
- 阶段一重点验证分类体系是否稳定、L1/L2分类是否可行
- 阶段二才建设轻量分类模型处理高频品类
- 阶段三才追求规模化自动分类
- 所有指标分阶段设定，不跳级

---

## 1. 方案比较（保留第一版评估结论）

简要回顾7种方案的适用性：

| 方案 | 评分 | 本项目适用阶段 |
|------|------|-------------|
| A: 关键词/规则匹配 | ⭐⭐ | 作为Layer 0前置，不是主方案 |
| B: 商品词典+品类映射 | ⭐⭐⭐ | 实体抽取的辅助工具 |
| C: TF-IDF+SVM/LR | ⭐⭐⭐ | 快速baseline，不适合上线 |
| D: MacBERT微调 | ⭐⭐⭐⭐ | 阶段二开始的主力模型 |
| E: LLM直接分类 | ⭐⭐⭐⭐ | 阶段一冷启动主方案 |
| F: LLM标注+小模型线上 | ⭐⭐⭐⭐⭐ | 推荐路线 |
| G: 端到端混合 | ⭐⭐⭐⭐⭐ | 最终目标架构 |

## 2. 推荐方案：三阶段渐进式混合方案

### 2.1 阶段一：LLM主导 + 规则辅助 + 人工校验（第1-4周）

**定位：验证分类体系，积累标注数据，不做自动上线**

```
投诉文本进入
    │
    ▼
┌─────────────────────────────┐
│ Step 1: LLM实体抽取+分类     │  ← 一次LLM调用同时完成实体抽取和品类分类
│ - 输入：清洗后文本+品类体系  │
│ - 输出：实体列表+L1/L2分类   │
│ - 覆盖率：100%              │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ Step 2: 规则校验             │
│ - 品类编码一致性校验         │
│ - 实体-品类逻辑一致性        │
│ - product_service_type合理性 │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────────────────────┐
│ Step 3: 置信度分档           │
│ - ≥0.80: 高置信度（可抽样）  │
│ - 0.60-0.80: 中置信度（建议复核）│
│ - <0.60: 低置信度（必须复核） │
└─────────────┬───────────────┘
              │
              ▼
       ┌──────┴──────┐
       │  人工校验    │  ← 抽样10-20%进行质量检查
       │  修正+标注   │
       └──────┬──────┘
              │
              ▼
       ┌──────────────┐
       │  标注数据积累  │  ← 目标：≥3000条有效标注
       └──────────────┘
```

**关键技术决策：**
- 实体抽取和品类分类用**一次LLM调用**完成（减少成本）
- 分类目标：**L1+L2为主，不强制L3**
- 人工校验量：**500-1000条抽样校验**（不是全量）
- 标注产出：≥3000条LLM分类+人工校验的数据

**阶段一验收指标（现实主义版）：**

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| L1分类准确率（人工校验subset） | ≥80% | 人工标注 vs LLM分类 |
| L2分类准确率（人工校验subset） | ≥70% | 人工标注 vs LLM分类 |
| 实体抽取成功率 | ≥85% | 成功抽到≥1个商品/服务实体的比例 |
| 人工校验量 | 500-1000条 | 覆盖所有L1品类 |
| 标注数据积累 | ≥3000条 | LLM分类+人工校验的总量 |
| 标注一致性（双人标注） | Kappa≥0.70 | 双人标注对比 |
| "其他/待人工复核"占比 | ≤30% | LLM分类中归入此类的比例 |

**不要追求的目标（阶段一）：**
- ❌ L3分类准确率（L3还没建）
- ❌ 自动分类覆盖率（还在冷启动，不自动分流）
- ❌ 线上推理延迟（还没上线）
- ❌ 小模型训练（数据还不够）

### 2.2 阶段二：词典+轻量模型+LLM兜底（第5-8周）

**定位：建设可自动分流的轻量系统，降低LLM调用成本**

```
投诉文本进入
    │
    ▼
┌─────────────────────────────┐
│ Layer 0: 商品词典精确匹配    │
│ - 覆盖率目标：30-40%        │
│ - 命中→置信度0.90→直接输出  │
└─────────────┬───────────────┘
              │ 未命中
              ▼
┌─────────────────────────────┐
│ Layer 1: 轻量模型分类        │
│ - 模型：MacBERT-base        │
│ - 分类目标：L1+L2（约135类，20业务L1+1兜底L1+114 L2）│
│ - 训练数据：≥3000条标注     │
│ - 置信度≥0.75→直接输出      │
│ - 覆盖率目标：40-50%        │
└─────────────┬───────────────┘
              │ 置信度<0.75或模型不确定
              ▼
┌─────────────────────────────┐
│ Layer 2: LLM兜底             │
│ - 处理：长尾品类/复杂语义     │
│ - 置信度≥0.80→输出          │
│ - 覆盖率目标：10-20%        │
└─────────────┬───────────────┘
              │ 置信度<0.80
              ▼
┌─────────────────────────────┐
│ Layer 3: 人工复核            │
│ - 覆盖率目标：15-20%        │
└─────────────────────────────┘
```

**模型训练要点：**
- 分类头数量：L1(20个业务L1+1个兜底L1=21类)+L2(114类)=约135个输出类，不是600类
- 训练时L1和L2两个独立分类头（分层softmax）
- L2分类头的预测受L1约束（先预测L1，然后在L1对应的L2中选）
- 使用Focal Loss处理类不平衡

**阶段二验收指标：**

| 指标 | 目标值 |
|------|--------|
| L1分类准确率（小模型） | ≥85% |
| L2分类准确率（小模型） | ≥75% |
| 自动分类覆盖率（不进入LLM+人工） | ≥60% |
| 人工复核占比 | ≤20% |
| LLM调用占比 | ≤20% |

**不能做的事：**
- ❌ 追求L3全覆盖（标注数据还不足）
- ❌ 承诺所有品类自动分类达标（长尾品类准确率还会低）

### 2.3 阶段三：小模型稳定运行+LLM长尾（第9周+）

**定位：规模化自动分类，持续优化**

```
投诉文本进入
    │
    ▼
┌─────────────────────────────┐
│ Layer 0: 词典(覆盖40%+)     │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│ Layer 1: 小模型(覆盖45%+)    │ ← 持续增量训练
│ - 高频品类 L1+L2自动分类    │
│ - 试点L3分类（高频品类）     │
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│ Layer 2: LLM(覆盖<10%)      │ ← 仅处理长尾和复杂case
└─────────────┬───────────────┘
              ▼
┌─────────────────────────────┐
│ Layer 3: 人工(<10%)         │
└─────────────────────────────┘
```

**阶段三验收指标：**

| 指标 | 目标值 |
|------|--------|
| L1准确率 | ≥92% |
| L2准确率 | ≥85% |
| 高频L3试点准确率 | ≥80% |
| 自动覆盖率 | ≥80% |
| 人工复核占比 | ≤10% |

## 3. 模型评估方案

### 3.1 评估指标（分阶段）

| 指标 | 阶段一 | 阶段二 | 阶段三 |
|------|--------|--------|--------|
| L1准确率 | ≥80% | ≥85% | ≥92% |
| L2准确率 | ≥70% | ≥75% | ≥85% |
| L3准确率 | — | — | ≥80%（高频试点） |
| 实体抽取成功率 | ≥85% | ≥90% | ≥93% |
| 自动覆盖率 | — | ≥60% | ≥80% |
| 人工复核占比 | 全部人工校验 | ≤20% | ≤10% |

### 3.2 评估方法

1. **留出验证集：** 10-15%标注数据不参与训练，仅用于评估
2. **人工抽样评估：** 每周从自动分类结果中抽200条检查
3. **品类维度分析：** 分别统计高/中/低频品类的准确率
4. **标注一致性评估：** 阶段一必须做双人标注一致性检查

### 3.3 标注一致性（重中之重）

阶段一最重要的评估指标不是模型准确率，而是**人工标注一致性**。

- 如果两个标注员对同一条投诉的分类一致率只有60%，那模型做到80%就是不可能的
- 需要先通过标注指南+样本讨论把一致性提到≥80%
- 只有人工一致性达标后，模型评估才有意义

## 4. 成本估算（现实主义版）

### 4.1 阶段一成本

| 项目 | 估算 |
|------|------|
| LLM调用（3000条×2轮） | ~6000次调用，约¥30-60 |
| 人工校验（500-1000条） | ~5-10人天 |
| 词典建设 | ~3-5人天 |
| **阶段一总成本** | **约10-15人天 + ¥30-60 API费用** |

### 4.2 阶段二成本

| 项目 | 估算 |
|------|------|
| GPU训练（云GPU按量） | ~¥100-300 |
| LLM兜底调用（约20%流量） | 视数据量而定 |
| 人工复核（约20%流量） | 视数据量而定 |

### 4.3 长期成本趋势

- LLM调用占比：100% → 20% → <10%
- 人工复核占比：100%校验 → 20%复核 → <10%复核
- 每万条投诉处理成本：从¥200+到¥30-
